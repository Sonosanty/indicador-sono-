1.  QUE ES OPENCLAW?

OpenClaw es un asistente de inteligencia artificial de codigo abierto que funciona directamente en tu ordenador. Tus datos no salen de tu PC salvo cuando llama a la API del modelo de IA configurado.

Que puede hacer en Windows

Chatear con IA desde el navegador en http://127.0.0.1:18789

Conectarse con Telegram, WhatsApp, Discord, Slack

Ejecutar tareas en tu PC con tu aprobacion explicita

Usar modelos: Gemini (gratis), Claude, GPT-4, Nvidia, o modelos locales via Ollama

Recordar contexto entre sesiones con MEMORY.md y AGENTS.md

Buscar informacion en la web en tiempo real (skill Perplexity Search via Playwright)

Ejecutar skills personalizados escritos en Node.js

[ INFO ]

OpenClaw tiene DOS partes:

  El GATEWAY: servidor local en http://127.0.0.1:18789 (solo tu equipo puede acceder)

  El CLI: el programa que controlas desde PowerShell

Inicias el Gateway con start-openclaw.bat, abres el navegador y chateas.

6.  CONFIGURACION DE SEGURIDAD

[ CORRECTO ]

El Gateway solo escucha en 127.0.0.1 (loopback). Nadie excepto tu propio PC puede conectarse. No es accesible desde Internet ni desde tu red WiFi.

Las 4 capas de proteccion

Capa

Mecanismo

Capa 1

gateway.bind = "loopback" en openclaw.json — Puerto invisible desde la red

Capa 2

Reglas de Firewall de Windows — Bloqueo adicional puertos 18789/18791

Capa 3

Sin Tareas Programadas ni Servicios — Tu decides cuando arranca

Capa 4

Backups automaticos de configuracion — Recuperacion rapida

Verificar el puerto con netstat

netstat -an | findstr ":18789"

Lo que ves

Significado

127.0.0.1:18789  LISTENING

CORRECTO — IPv4 loopback, normal y seguro

[::1]:18789  LISTENING

CORRECTO — IPv6 loopback, equivalente a 127.0.0.1

0.0.0.0:18789  LISTENING

PELIGROSO — Expuesto a toda la red; corregir en openclaw.json

Opcion 1: Cierra la ventana negra del terminal

NOTA IMPORTANTE — ConvertFrom-Json en PS5:

En PowerShell 5.1, ConvertFrom-Json NO acepta el parametro -Depth.

Ese parametro fue introducido en PS 6.2+.

Usa siempre: $cfg = $jsonRaw | ConvertFrom-Json  (sin -Depth)

El -Depth solo va en ConvertTo-Json cuando serializas.

15.  SOLUCION AL PROBLEMA GOOGLE:MANUAL

[ PELIGRO ]

PROBLEMA CRITICO IDENTIFICADO:

En los logs aparece repetidamente:

  [agent/embedded] Profile google:manual timed out. Trying next account...

  [agent/embedded] [llm-idle-timeout] google/gemini-2.5-flash produced no reply

Esto provoca que cada request tarde 2-5 MINUTOS y falle al final.

El timeout no es de Google — es que OpenClaw usa el perfil "manual"

que requiere token de sesion web en vez de API key directa.

Causa raiz

Cuando configuras Gemini via CLI con la opcion "Google AI Studio" + "Google auth method: Back" (cancelando), OpenClaw registra un perfil google:manual que intenta autenticarse via sesion de navegador en vez de API key. Ese token de sesion expira rapidamente y causa los timeouts.

Solucion: reemplazar google:manual por API key directa

Script PS5 completo para corregir el perfil:

$cfgPath = "$env:USERPROFILE\.openclaw\openclaw.json"

$cfg = Get-Content $cfgPath -Raw | ConvertFrom-Json

 

# Eliminar el perfil manual problematico

if ($cfg.PSObject.Properties["auth"] -and $cfg.auth.PSObject.Properties["profiles"]) {

    $toRemove = @()

    foreach ($p in $cfg.auth.profiles.PSObject.Properties) {

        if ($p.Name -like "google:manual*") { $toRemove += $p.Name }

    }

    foreach ($n in $toRemove) {

        $cfg.auth.profiles.PSObject.Properties.Remove($n)

        Write-Host "Eliminado perfil: $n" -ForegroundColor Yellow

    }

}

 

# Agregar API key directa de Gemini

if (-not $cfg.PSObject.Properties["env"]) {

    $cfg | Add-Member -NotePropertyName "env" -NotePropertyValue ([PSCustomObject]@{}) -Force

}

$cfg.env | Add-Member -NotePropertyName "GEMINI_API_KEY" -NotePropertyValue "AIzaSy-TU-KEY-AQUI" -Force

 

# Guardar y reiniciar

$cfg | ConvertTo-Json -Depth 20 | Set-Content $cfgPath -Encoding UTF8

Write-Host "Config actualizado" -ForegroundColor Green

 

# Matar node y limpiar locks antes de reiniciar

taskkill /F /IM node.exe 2>$null

Start-Sleep -Seconds 3

Get-ChildItem "$env:USERPROFILE\.openclaw\agents" -Recurse -Filter "*.lock" -ErrorAction SilentlyContinue | Remove-Item -Force

 

# Arrancar limpio

& "C:\OpenClaw\start-openclaw.bat"

Verificar que el perfil manual fue eliminado

16.  OPTIMIZAR SESSION-RESOURCE-LOADER

[ ATENCION ]

SEGUNDO PROBLEMA CRITICO:

En los logs aparece: session-resource-loader: 5857ms, 17829ms, 16787ms

Y: low context window: google/gemini-2.5-flash ctx=60000 (warn<209715)

El contexto activo esta en 60.000 tokens cuando deberia ser 1.000.000.

Esto hace que OpenClaw tenga que recargar recursos en cada request.

Causa: contextTokens demasiado bajo

El valor ctx=60000 indica que agents.defaults.contextTokens esta configurado en 60000 (o que el config se redujo por el size-drop protection). Gemini 2.5 Flash soporta hasta 1 millon de tokens.

Correccion directa via PowerShell (sin openclaw config set)

[ INFO ]

El comando "openclaw config set agents.defaults.contextTokens 1000000" puede ser rechazado por la proteccion size-drop del config. Usa este metodo directo:

$cfgPath = "$env:USERPROFILE\.openclaw\openclaw.json"

$cfg = Get-Content $cfgPath -Raw | ConvertFrom-Json

 

# Asegurar que agents.defaults existe

if (-not $cfg.PSObject.Properties["agents"]) {

    $cfg | Add-Member -NotePropertyName "agents" -NotePropertyValue ([PSCustomObject]@{}) -Force

}

if (-not $cfg.agents.PSObject.Properties["defaults"]) {

    $cfg.agents | Add-Member -NotePropertyName "defaults" -NotePropertyValue ([PSCustomObject]@{}) -Force

}

 

# Subir contextTokens a 1 millon

if ($cfg.agents.defaults.PSObject.Properties["contextTokens"]) {

    $cfg.agents.defaults.contextTokens = 1000000

} else {

    $cfg.agents.defaults | Add-Member -NotePropertyName "contextTokens" -NotePropertyValue 1000000 -Force

}

 

# Guardar con escritura directa (bypassa la proteccion size-drop)

$newJson = $cfg | ConvertTo-Json -Depth 20

[System.IO.File]::WriteAllText($cfgPath, $newJson, [System.Text.UTF8Encoding]::new($false))

Write-Host "contextTokens = 1.000.000 aplicado" -ForegroundColor Green

Limpiar sesiones bloqueadas

El session-resource-loader se ralentiza cuando hay sesiones viejas con historial pesado. Limpia periodicamente:

# Limpiar sesiones huerfanas

openclaw sessions cleanup --store "$env:USERPROFILE\.openclaw\agents\main\sessions\sessions.json" --enforce --fix-missing

 

# Borrar fisicamente una sesion bloqueada especifica

# (reemplaza el UUID con el que aparece en tus logs)

Problema 4: Config rechazado (size-drop)

[ ATENCION ]

OpenClaw rechaza escrituras de config cuando el nuevo archivo es mucho mas pequeno que el actual (proteccion size-drop). Usa [System.IO.File]::WriteAllText() para bypassarla.

$cfgPath = "$env:USERPROFILE\.openclaw\openclaw.json"

$cfg = Get-Content $cfgPath -Raw | ConvertFrom-Json

# ... modifica $cfg ...

$newJson = $cfg | ConvertTo-Json -Depth 20

[System.IO.File]::WriteAllText($cfgPath, $newJson, [System.Text.UTF8Encoding]::new($false))
