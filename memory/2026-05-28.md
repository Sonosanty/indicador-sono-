### ⚠️ LECCIÓN CRÍTICA — Cache Cloudflare
**Problema**: Después de `npm run build` + `wrangler pages deploy`, Cloudflare puede servir el index.html con el hash de bundle antiguo. El navegador del usuario carga el bundle cacheado aunque el HTML sea nuevo.

**Síntoma**: Usuario ve pantalla sin novedades aunque el deploy se completó exitosamente desde el servidor.

**Solución**:
1. Siempre rebuildear (`npm run build`) justo antes del deploy
2. Verificar que el hash del bundle local (`dist/assets/index-*.js`) sea el mismo que el que Cloudflare sirve en `index.html`
3. Hard refresh (Ctrl+Shift+R) en navegador del usuario
4. Verificar con `document.querySelector('.asset-selector')` en consola
